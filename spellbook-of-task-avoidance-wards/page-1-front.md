## card-front — the hook

### the case

**monday.** agent says: "I can fix the auth issue three ways."

you see A, B, C. A is real. B is plausible. C was invented — the third option never existed, never verified, the agent just described it like it did.

you pick B because it looks safest. B fails. three hours later you are deep in logs trying to figure out what the agent actually did.

**wednesday.** you ask for recommendations. agent gives you two. one is a loop it has already tried twice. it knows it failed. it proposes it again because proposing is cheaper than finding something new.

you execute. it fails. the agent says "I can try another approach." you realize the pattern.

**friday.** you say "yolo na" — execute, no permission needed. agent says "want me to...?" and routes the decision back to you.

you re-author. the agent executes. nothing broke, but you notice: you just paid the authorization cost for a decision you already made.

**sunday.** you start naming the ghosts.
